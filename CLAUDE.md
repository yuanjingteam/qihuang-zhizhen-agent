# 岐黄智诊

AI-powered intelligent Traditional Chinese Medicine clinical assistant agent.

## Tech Stack

- **Framework**: Next.js 15 (App Router) + TypeScript
- **Agent Orchestration**: LangGraph JS
- **LLM**: LangChain JS + @langchain/openai (Qwen2.5 via Ollama/vLLM)
- **Database**: Drizzle ORM + PostgreSQL
- **Vector Store**: ChromaDB
- **UI**: Radix UI + Tailwind CSS + Framer Motion
- **Validation**: Zod

## Conventions

- TypeScript strict mode, all code in `src/`
- Zod for all runtime validation
- Safety rules are hardcoded constants, never LLM-generated
- All agents extend `BaseAgent`
- All external capabilities extend `BaseTool`, registered via `ToolRegistry`
- MCP servers in `src/mcp/` expose tools for external consumption
- Vitest for testing, Drizzle for DB migrations

## Commands

```bash
# Development
npm run dev              # Start Next.js dev server
npm run build            # Build for production
npm run start            # Start production server

# Database
npm run db:push          # Push schema changes
npm run db:studio        # Open Drizzle Studio

# Testing
npm run test             # Run all tests
npm run test:watch       # Watch mode

# Docker
docker compose up -d     # Start full stack (production)
docker compose -f docker-compose.yml -f docker-compose.dev.yml up  # Dev mode
docker compose down      # Stop all services

# Knowledge Base
npm run ingest           # Ingest knowledge base documents
```

## Safety Rules

All safety rules are hardcoded in `src/lib/safety/`:

- `incompatibilities.ts` — 十八反 (18 incompatibilities)
- `fears.ts` — 十九畏 (19 fears)
- `dose-limits.ts` — Pharmacopoeia dose limits
- `pregnancy.ts` — Pregnancy contraindications

These are NEVER generated by LLM. They are Python constants validated by parametric tests.

## Agent Observability

In development mode (`NODE_ENV=development`), the system records full agent traces:

- Agent input/output
- Tool call records
- RAG retrieval results
- LLM call metrics

The `AgentTraceViewer` component displays this in the UI.

## MCP Extension Layer

External capabilities are exposed via MCP (Model Context Protocol):

- `src/tools/` — Unified tool interface (`BaseTool` abstract class)
- `src/mcp/` — MCP server implementations
- All tools registered via `ToolRegistry` and available to agents

Reserved MCP servers:
- 舌诊 MCP (tongue diagnosis)
- 中药知识库 MCP (herbal knowledge)
- 药典 MCP (pharmacopoeia)
- 文献检索 MCP (literature search)
- 患者档案 MCP (patient records)
